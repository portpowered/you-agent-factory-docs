## Codebase Patterns
- Documentation pages: structure in `page.mdx`, prose in colocated `messages/<locale>.json`, empty `assets.json` (`{}`) unless teaching media is required; registry record at `src/content/registry/documentation/<slug>.json` with id `documentation.<slug>`.
- Fumadocs frontmatter still needs non-empty `title` / `description` even when reader copy is message-backed; docs shell owns the visible h1 — do not add in-body `# <T k="title" />`.
- Draft/scaffold documentation section bodies must be non-empty (validate-data treats empty `sections.*.body` as missing MDX keys).
- Colocate page render proofs as `<slug>-page.test.tsx` under the page bundle; assert identity, section headings, no Model Atlas / reader-shortcut / in-body h1 title duplicate.
- Browser verify after content edits: `bun run build` then `bun run start` on a unique port (3100–3999), `curl --max-time 10`, kill the server before exit — `start` serves the last production build.
- Ground packaged-docs claims in live CLI: `you docs` (index) / `you docs <topic>` / `you docs --help` (“Print packaged markdown reference topics from the installed binary”).
- Ground packaged-factories claims in live CLI: `you run --help` (`--named` resolution + built-in materialization) and `you docs packaged-goal|packaged-fusion|packaged-tts` (canonical `@you/*` names, URL-encoded disk paths, editability).
- Packaged-factory resolution/URL-encoding detail lives in packaged-topic “Where the factory materializes” sections (`you docs packaged-goal` / `packaged-tts`), not only in `you run --help`.
- Selectable CLI examples: put commands in MDX fenced `bash` blocks with message-backed labels; keep discovery/mechanism callouts under Key Concepts so How To Use stays command-focused.
- Browser HTML may include Fumadocs TOC chrome (“On this page”); assert page-meta bans against message bodies / section nodes, not raw full-document HTML alone.
- After adding a new page bundle, run `make validate-data` (or `prepare:content-runtime` + `fumadocs-mdx`) before `source.getPage` tests — otherwise fumadocs source can still return `undefined` until MDX regen.
- Documentation→documentation related links need page-local `<LocalizedLinkList>` (RelatedDocs omits documentation kinds); keep registry `relatedIds` aligned for metadata. Packaged-documents → sidebar `cli`; packaged-factories → `configuration`.
- Locale stubs: copy `en.json` key shape to `ja` / `zh-CN` / `vi`, then `bun run generate:shipped-localized-docs` and update `shipped-localized-docs.server.test.ts` committed-tree expectations alphabetically.
- Multi-page documentation lanes that need both `FACTORY_DOCUMENTATION_SIDEBAR_GROUP_BY_SLUG` membership and locale-shipping: audit page-owned (`keep-routine`), then locale-only and sidebar-only each with `--exception-reason` (`declare-exception`). Combining both shared paths in one `--files` list still redirects; document both reasons in the PR conversation without redesigning the classifier.
