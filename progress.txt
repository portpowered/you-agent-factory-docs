## Codebase Patterns
- Canonical content types and validation live in `src/lib/content/`; import from `@/lib/content`. Keep this separate from docs-shell UI copy in `src/lib/shell.ts`.
- `validateContentMetadata()` returns `{ ok: true, record }` or `{ ok: false, errors }` with reviewer-verifiable `{ field, message }` entries instead of throwing.
- Canonical ids use `{kind}/{slug}` and must match the declared `kind` and `slug`; route paths are derived per public content kind.
- Locale tags use BCP 47-style `en` or `en-US`; `canonicalLocale` must be included in `availableLocales`.
- Root `Makefile` targets (`setup`, `check`, `test`, `build`) mirror `package.json` scripts; component tests use `@testing-library/react` with happy-dom preloaded from `bunfig.toml`.
- Starter content fixtures live under `src/content/{docs,blog,glossary,comparisons,references}/{slug}/{locale}.mdx`; use `validateStarterContent()` or `loadStarterContentRecords()` to project validated canonical records from frontmatter plus directory context.
- Docs-shell navigation is projected via `projectDocsShellNavigation()` / `loadDocsShellNavigation()` into `DocsShellNavigationInput`; `DocsShell` consumes that input via props instead of hand-maintained nav entry constants.
