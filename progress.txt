## Codebase Patterns
- Root contributor commands live in the `Makefile` (`setup`, `check`, `test`, `build`) and mirror `package.json` scripts.
- Bootstrap scaffold patterns are documented in `docs/internal/processes/bootstrap-static-export-foundation-relevant-files.md`; reconciliation lane notes live in `docs/internal/processes/reconcile-bootstrap-foundation-baseline-relevant-files.md`.
- Command-path proof uses `tests/helpers/make-target.ts` (`spawnSync` on `make <target>`); assert on combined stdout+stderr because `bun test` reports on stderr.
- When a test invokes `make test`, set `VERIFYING_MAKE_TEST=1` in the child env and skip the recursive assertion when that env var is present.
- Component behavior tests use `@testing-library/react` with `next/link` mocked via `tests/helpers/mock-next-link.tsx`.
- App Router pages are thin wrappers around shell components (`src/app/page.tsx`, `src/app/docs/page.tsx`).
- The bootstrap foundation was merged to `main` via PR #1 (`4ee2134`); the reconciled tree matches `bootstrap-static-export-foundation` with no scaffold divergence.
- Browser verification for the static export reuses `tests/helpers/static-export-server.ts` (symlink mount under the GitHub Pages base path) with Playwright Chromium; static-export links use trailing slashes so href assertions should allow an optional `/` suffix.
- Reconciliation divergence is recorded in `docs/internal/processes/reconcile-bootstrap-foundation-baseline-divergence.md`; keep that file narrow to reconciliation-relevant differences and link it from the lane relevant-files doc.
