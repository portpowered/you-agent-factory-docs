# problem

sometimes we have a paper and we want to reference it, but the paper is fairly verbose. 

# Solution

this paper is how any agent that is responsible for reading a paper supposed to break down the task breakdown for a paper. 

# Overall

Given a paper, an agent should read it, break it down into its constituent components. 

1. did the paper introduce a new model? if so we should decompose it to a new page for a model.
2. did the paper introduce a new concept? if so we should add a new concept. 
3. did the paper introduce a new type of module? we should add it. 
4. we should add a new page for the paper overall as well. 

For each component, we should add the page in such a way that we conform to the specification of the page/template.

# structuring

- each page(model/module/concept/training/etc) should expose itself as an isolated paper. 

# planning
we recommend that you create an independent work batch of items and submit the batch as a group.

The individual modules/models/training/etc should be made independent and not depend directly on the paper. 
The individual ideas should be submitted with a reference to the paper associated but the ideas MUST try to write as if its independent of the paper and the rest of the group. 

When introducing new modules/training regimes/wtv. you should consider that the idea for the plan should be an emphasis of the value that it introduces,  and that we should focus on the rendering of equations/graphs that present this emphasis.
