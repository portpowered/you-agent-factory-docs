# Agent

An agent in You Agent Factory is a named worker inside a workflow. It receives instructions, can use the tools and files the workflow gives it, and returns an output that another step can inspect, approve, or reuse.

## What the term means here

In this project, an agent is not the whole factory and it is not a vague synonym for "AI." It is one bounded participant with a clear role, such as drafting a change, reviewing a diff, collecting evidence, or preparing a follow-up task. The workflow decides when that agent runs, what context it sees, and what happens with its output.

## Why it matters in You Agent Factory

The distinction matters because You Agent Factory is built around repeatable process design, not one large prompt. Teams break recurring work into smaller responsibilities so each agent can be inspected, retried, or replaced without rewriting the whole workflow. That makes approvals, logs, and failure handling easier to reason about.

## How to read the term as a first-time evaluator

When the docs say "agent," read it as a scoped unit of work with a specific contract. A useful agent should have a narrow job, explicit inputs, and an output that the rest of the workflow can validate. If a step has no clear owner or no clear handoff, it is usually a sign that the workflow still needs better structure rather than more agent autonomy.
