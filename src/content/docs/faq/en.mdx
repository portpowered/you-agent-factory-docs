# FAQ

Practical answers for first-wave evaluators who want to decide whether You Agent Factory fits recurring engineering work.

## What is You Agent Factory for?

You Agent Factory is for turning recurring development work into reusable, inspectable agent workflows instead of handling each run as a fresh prompt.

Use it when the same engineering tasks keep coming back, such as documentation updates, review passes, release checks, or follow-up work after incidents, and you want a defined process that can be repeated and improved over time.

## Who is it for first?

It is for engineering teams that already work from repositories, pull requests, local tools, and explicit review steps, and want those recurring workflows to become easier to rerun, inspect, and refine.

Teams get the most value first when they already know the task pattern they want to repeat and need a better way to keep the workflow consistent across runs.

## How is it different from one-off chat usage?

One-off chat is useful for isolated questions or ad hoc tasks, but it does not give you a durable workflow definition that your team can run again with the same structure.

You Agent Factory focuses on repeatability. The workflow, inputs, outputs, and checkpoints live in a form that engineers can inspect, review, compare, and update instead of depending on one successful prompt session.

## How is it different from running a single agent ad hoc?

Running a single agent ad hoc can help with a task once, but it still leaves the surrounding process implicit.

You Agent Factory is designed for the broader workflow around the task: which steps run, where tools or files are involved, what gets handed off, and how the same pattern becomes reusable for the next run.

## When is it a good fit?

It is a good fit when the work is recurring enough that the team benefits from a shared, inspectable workflow instead of relying on memory, personal prompting habits, or copied chat transcripts.

If the task is still highly exploratory or one-time, a direct chat or a simple single-agent run may be the faster starting point.
