# Installation

Install You Agent Factory locally using the repository's supported Bun and Make workflow before moving into the first workflow run.

## Prerequisites

Start with Bun 1.1 or newer available on your machine because the repository uses Bun for dependency installation, scripts, and test execution.
Clone the repository and work from its root so the documented Make targets run against the same command path used by local contributors and CI.

## Install the repository

Run `make setup` from the repository root to install or refresh dependencies.
This target is the supported entrypoint for initial setup and it delegates to `bun install` instead of requiring a separate manual package-manager path.

## Validate the install

Run `make check`, `make test`, and `make build` after setup to verify that the local install matches pull request validation.
These are the authoritative repository-root commands for contributor and PR parity, so the same typecheck, lint, automated tests, and static export checks fail in both places without switching to a CI-only command surface.

## What success looks like

Treat the install as successful when `make check`, `make test`, and `make build` finish without failures from the repository root.
If you want the broader local-only foundation sweep before review, run `make quality-gate` afterward; it intentionally goes beyond PR validation by adding localization, content, accessibility, and static-export contract checks.

## Continue into quickstart

Continue to Quickstart once the repository installs cleanly and the validation path passes so your first workflow run starts from a known-good local setup.
